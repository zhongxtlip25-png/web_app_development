# 產品需求文件 (PRD) - 食譜收藏系統

## 1. 專案概述
- **背景與動機**：平時看到喜歡的食譜經常散落在不同網站或是社群軟體中，不易重複查閱或依據現有的食材來發想能煮什麼。因此需要一個可以集中管理食譜，並輔助日常採買與烹飪計畫的系統。
- **目標用戶**：對烹飪有興趣的個人使用者（如自己）。
- **核心價值主張**：提供一個簡單、直覺的個人專屬食譜資料庫，整合食材採買與備餐日曆，讓選擇三餐、採買食材到實際烹飪的流程更為順暢。

## 2. 功能需求
系統提供以下主要功能，並以使用者故事來呈現：

- **新增食譜**
  - 作為使用者，我希望可以新增與編輯食譜（包含食材、份量、作法步驟與照片），以便把喜歡的料理配方存檔備查。
- **尋找食譜**
  - 作為使用者，我希望可以透過關鍵字、分類或標籤搜尋食譜，以便快速找到適合當下的料理。
- **根據食材推薦食譜**
  - 作為使用者，我希望系統能根據我輸入現有的食材，去自動推薦或篩選出相關食譜，以便消耗冰箱內的剩餘食材。
- **購物清單自動生成**
  - 作為使用者，我希望在選定某幾篇食譜後，系統能將所需材料自動匯總並產生購物清單，以便去超市採買時不會有遺漏。
- **烹飪計畫與日曆**
  - 作為使用者，我希望可以將想做的食譜排程到日曆上（如本週三晚餐），以便規劃一週的飲食計畫與便當菜色。

## 3. 非功能需求
- **技術限制**：
  - 後端：Python + Flask
  - 前端網頁：HTML + CSS + Vanilla JS (使用 Jinja2 模板引擎)
  - 資料庫：SQLite (適合單人系統，輕量且易於維護備份)
- **效能與安全考量**：
  - 確保頁面載入速度與查詢時間在合理範圍（2秒以內）。
  - 對於表單輸入做好基礎的欄位驗證設定，並防護基本的 SQL Injection 與 XSS。
  - 需要一組簡單的管理員帳號機制來保護系統不被他人隨意修改。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必要功能)
- 新增、編輯、查看、刪除食譜。
- 基礎的食譜列表顯示與關鍵字搜尋功能。

### Should Have (重要功能)
- 根據輸入的食材關鍵字來篩選合適的食譜。
- 購物清單：將指定食譜內的食材直接轉成待買清單字串或勾選表。

### Nice to Have (加分功能)
- 烹飪日曆排程：拖曳或設定食譜到日曆的不同日期上。
- 匯入網址功能（自動爬取外部網頁食譜內容）。
- 食材庫存管理機制。
